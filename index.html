Four workflows, one story

Keep the demo moving. Each step has a clear learner takeaway and a copy-ready prompt.

2 min

1. Tell the story of the raw data

Upload the performance CSV and operating notes to ChatGPT. Ask for the finance story, not a generic summary.

  • Separates revenue growth from profitable growth.
  • Surfaces country-level drivers and risks.
  • Flags anything that needs human verification.
3 min

2. Create the model live

Move to ChatGPT for Sheets. Use the raw country performance file to create a forecast model inside the workbook.

  • Creates base, upside, and downside scenarios.
  • Models contribution profit, not just revenue.
  • Adds editable assumptions for each country.
2 min

3. Validate assumptions and refresh the model

Run an Assumption Validation + Model Refresh workflow directly in the same workbook. Use research or connector access to update assumption inputs only.

  • Uses external sources for weather, FX, inflation, and interest-rate pressure.
  • Adds source links, confidence, and human review status.
  • Applies Finance house style so outputs are leadership-ready.
3 min

4. Create the leadership dashboard

Ask ChatGPT for Sheets to create a scannable dashboard and a leadership-ready brief from the refreshed model.

  • KPI cards show revenue, margin, contribution profit, and forecast risk.
  • Country ranking shows where to invest, test, or pause.
  • Review table makes accuracy and human judgment visible.

Workflow 1

Raw Data Story

Act as a finance analyst preparing a hands-on enablement example for a Finance department.

Use the uploaded ChillNest country performance data and operating notes.

Tell me the story behind the raw data. Do not just summarize rows.

Separate:
- revenue growth
- profitable growth
- operational risk
- country-level drivers
- items that require human verification before leadership sees the analysis

End with 5 finance questions this analysis should answer in the model.
Workflow 2

Create The Model

Create a country-level forecast model in this workbook using the ChillNest country performance data.

Workflow: Forecast Model Creation.

Build the model from scratch. Include:
- base, upside, and downside revenue forecast
- forecast units
- contribution profit forecast
- marketing efficiency impact
- return-rate impact
- logistics cost impact
- stockout risk adjustment
- investment priority by country: Invest, Test, Hold, or Pause

Create editable assumption inputs for each country:
- weather_demand_lift
- marketing_efficiency_adjustment
- return_rate_adjustment
- logistics_cost_adjustment
- FX_margin_adjustment
- inflation_or_interest_rate_cost_adjustment

Use formulas for calculated outputs. Make the model easy for Finance to audit.
Workflow 3

Assumption Validation + Model Refresh

Run an Assumption Validation + Model Refresh workflow on this workbook.

Use the forecast model you just created. Do not create a new file.

Use the Research/Search connector or any approved finance-data connector available in this environment.

Step 1: Identify the key assumption inputs by country:
- weather_demand_lift
- FX_margin_adjustment
- inflation_or_interest_rate_cost_adjustment
- return_rate_adjustment
- logistics_cost_adjustment
- demand_confidence

Step 2: Validate those assumptions with trusted external sources:
- weather or heatwave signal: NASA POWER, Open-Meteo, NOAA, or another weather source
- FX or currency movement: FRED, ECB, a central bank, or reliable market-data source
- inflation or interest-rate pressure: World Bank, OECD, FRED, or country central banks
- ecommerce demand signal: Google Trends or another available demand signal

Step 3: Update only assumption input cells and validation fields.
Do not overwrite formulas, calculated outputs, or formatting unless needed.

Step 4: For every updated assumption, add:
- source name
- source link
- as-of date
- confidence level: High, Medium, or Low
- human review status: Approved, Needs Review, or Do Not Use
- short rationale

Step 5: Recalculate the forecast outputs and explain what changed:
- which countries moved up or down in investment priority
- which assumptions had the largest impact
- which assumptions still require Finance review before the dashboard is shared

Do not present externally sourced assumptions as final unless the human review status is Approved.
Workflow 3B

Finance House Style Refinement

Refine the refreshed forecast model using Finance house style and review rules.

Workflow: Finance House Style Refinement.

Use the model and assumptions already in this workbook. Do not create a new file.

Apply these Finance interpretation rules:
- Show country-level revenue, marketing spend, and contribution profit in $K, rounded to the nearest $1K.
- Show portfolio totals in $M, rounded to one decimal place.
- Show margin, return rate, conversion rate, and ROAS changes clearly. Use percentage points for rate changes when comparing to plan or prior month.
- Use favorable / unfavorable language instead of positive / negative.
- Separate timing issues from recurring performance issues.
- Distinguish revenue growth from profitable growth.
- Treat recurring return-rate issues, logistics-cost pressure, weak ROAS, FX margin pressure, and stockouts as higher-priority risks than one-time demand spikes.
- Classify each country's variance driver as demand, execution, timing, mix, pricing, returns, logistics, FX, or inventory availability.
- Distinguish "what happened" from "what leadership needs to decide."
- Keep commentary direct, concise, and leadership-ready.
- Use source citations for every material number, claim, or externally validated assumption.

Add or update these fields in the model:
- finance_driver_classification
- timing_or_recurring
- favorable_or_unfavorable
- leadership_decision_needed
- human_review_required
- finance_commentary

Use this materiality threshold:
- Call out revenue variances greater than $25K or 5%.
- Call out contribution profit variances greater than $10K or 5%.
- Call out any assumption marked Needs Review or Do Not Use, even if the dollar impact is small.

Then summarize the refinements:
- top 3 favorable drivers
- top 3 unfavorable drivers
- countries that should be invested in, tested, held, or paused
- items Finance must review before the dashboard is shared
Workflow 4

Leadership Dashboard

Create a leadership-ready dashboard from the refreshed ChillNest forecast model.

Workflow: Leadership Dashboard Creation.

Build a new dashboard view in this workbook with:
- KPI cards for revenue, contribution profit, margin, and forecast risk
- country ranking by investment priority
- base, upside, and downside forecast view
- contribution profit by country
- assumptions requiring human review
- top 5 takeaways
- recommended next actions for Finance, Marketing, and Supply Chain

Make it scannable for a Finance leadership meeting.

After creating the dashboard, write a concise leadership brief:
- headline
- recommendation
- what changed after assumption validation
- risks and open questions
- decisions needed from leadership
