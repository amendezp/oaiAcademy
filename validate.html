Prompt 1

Assumption Validation + Model Refresh

Run an Assumption Validation + Model Refresh workflow on this workbook.

Workflow: ChatGPT for Sheets Assumption Validation + Model Refresh using the Research/Search connector. If an approved finance-data connector is available, use it for market, FX, rate, or inflation data. Do not create a new file.

Use the DCF model you just created. Focus on the assumptions most likely to move valuation in a 10-minute demo:
- weather_demand_lift for the largest revenue countries or regions
- FX_margin_adjustment for non-US markets with margin pressure
- inflation_or_interest_rate_cost_adjustment for logistics, working capital, and discount-rate context
- regional_trend_adjustment using the monthly regional performance file
- free_cash_flow_conversion_adjustment using the historical cash flow statement and working-capital trend
- discount_rate and terminal_growth_rate reasonableness

Step 1: Locate the current assumption input cells and review fields. Do not overwrite formulas, calculated outputs, or formatting.

Step 2: Validate assumptions with trusted sources:
- heat or weather signal: NASA POWER, Open-Meteo, NOAA, or another credible weather source
- FX or currency movement: FRED, ECB, country central bank, or reliable market-data source
- inflation or interest-rate pressure: World Bank, OECD, FRED, or country central bank
- demand signal: Google Trends or another available ecommerce demand signal
- internal trend: compare June performance to the monthly regional performance file before treating demand as durable
- cash conversion: compare inventory investment, working capital, capex, operating cash flow, and free cash flow before treating growth as value-creating

Step 3: Update only assumption input cells and validation fields.

For every updated assumption, add:
- source name
- source link
- as-of date
- old value
- new value
- valuation impact direction: favorable, unfavorable, or neutral
- confidence level: High, Medium, or Low
- human review status: Approved, Needs Review, or Do Not Use
- short rationale

Step 4: Recalculate the forecast outputs and create a short change log:
- countries that moved up or down in investment priority
- regions that changed in forecast or DCF-readiness
- assumptions with the largest valuation impact
- assumptions that still require Finance review before the dashboard is shared

Do not present externally sourced assumptions as final unless the human review status is Approved.
Prompt 2

Finance House Style Refinement

Refine the refreshed DCF model using Finance house style and review rules.

Workflow: Finance House Style Refinement.

Use the model and assumptions already in this workbook. Do not create a new file.

Apply these Finance interpretation rules:
- Show country-level revenue, marketing spend, and contribution profit in $K, rounded to the nearest $1K.
- Show portfolio totals in $M, rounded to one decimal place.
- Show margin, return rate, conversion rate, and ROAS changes clearly. Use percentage points for rate changes when comparing to plan or prior month.
- Use favorable / unfavorable language instead of positive / negative.
- Separate timing issues from recurring performance issues.
- Distinguish revenue growth from profitable growth.
- Treat recurring return-rate issues, logistics-cost pressure, weak ROAS, FX margin pressure, and stockouts as higher-priority risks than one-time demand spikes.
- Classify each country's variance driver as demand, execution, timing, mix, pricing, returns, logistics, FX, or inventory availability.
- Distinguish "what happened" from "what leadership needs to decide."
- Avoid generic commentary. Replace phrases like "strong performance" with the specific driver and implication.
- Separate actuals, forecast assumptions, and externally validated assumptions.
- Keep commentary direct, concise, and leadership-ready.
- Use source citations for every material number, claim, or externally validated assumption.

Add or update these fields in the model:
- finance_driver_classification
- timing_or_recurring
- favorable_or_unfavorable
- leadership_decision_needed
- human_review_required
- finance_commentary

Use this materiality threshold:
- Call out revenue variances greater than $25K or 5%.
- Call out contribution profit variances greater than $10K or 5%.
- Call out any assumption marked Needs Review or Do Not Use, even if the dollar impact is small.

Then summarize the refinements:
- top 3 favorable drivers
- top 3 unfavorable drivers
- countries that should be invested in, tested, held, or paused
- decisions leadership needs to make
- items Finance must review before the dashboard is shared
Human review moment: A source link is not the same as approval. Use the confidence and review-status fields to show where Finance still needs to make a judgment call.
