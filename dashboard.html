Create a finance dashboard

Turn the refreshed model into a dashboard a Finance reviewer can scan quickly. The goal is not decoration; the goal is fast decision support with visible review status.

Inputs
  • The refreshed DCF model
  • Validated assumptions and review statuses
  • Regional trends and country priority outputs
  • Any placeholder KPI definitions you want to teach
Outputs
  • KPI panels for Revenue, Gross Margin, Burn Rate, Runway, and Budget vs. Actual
  • At least three visuals for trends, scenarios, or valuation sensitivity
  • Clear labels for source-backed, calculated, validated, and placeholder metrics
  • A short leadership brief with risks, questions, and decisions needed

    Create the Dashboard

    Create a review-ready dashboard from the refreshed ChillNest DCF.

Use clean panels for:
- Revenue
- Gross Margin
- Burn Rate
- Runway
- Budget vs. Actual

Add views for regional trends, country priorities, DCF valuation, scenarios, assumption review, and top takeaways.

Include at least 3 visuals. Label each metric as source-backed, calculated, externally validated, or placeholder.

If a KPI is not meaningful, mark it "Not applicable." If data is missing, mark it "Placeholder - needs source data."

Finish with a concise leadership brief using favorable / unfavorable language: recommendation, risks, open questions, and decisions needed.
  3. Step 3

    Check every KPI label

    Confirm each metric is marked as source-backed, calculated, externally validated, placeholder, or not applicable.

  4. Step 4

    Review the brief

    Make sure the summary separates what happened, what changed in the model, and what leaders need to decide.

Workflow

Turn the refreshed model into a scannable dashboard.

The goal is fast decision support. Leaders should see the core KPIs, scenario impact, assumption status, risks, and decisions needed without hunting through the model.

Design check: The best dashboard is honest. Placeholder metrics are useful for layout, but they must be visually labeled so nobody mistakes them for actual company data.
